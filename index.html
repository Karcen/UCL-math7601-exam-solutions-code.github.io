(a) BLAS 基础线性代数子程序

BLAS（Basic Linear Algebra Subprograms）是一组标准化的线性代数运算接口，提供高度优化的底层数值计算例程。

级别 操作类型 示例 计算复杂度 内存访问
Level 1 向量-向量 点积、向量加法、范数 O(n) O(n)
Level 2 矩阵-向量 矩阵向量乘法 y=Ax O(n²) O(n²)
Level 3 矩阵-矩阵 矩阵乘法 C=AB O(n³) O(n²)

关键观察：Level 3的计算强度（计算量/内存访问量）为O(n)，远高于Level 1和Level 2。

计算强度定理

定义：计算强度（Computational Intensity）定义为浮点运算数量与内存访问数量的比值。 $$\text{计算强度} = \frac{\text{浮点运算次数}}{\text{内存访问次数}}$$

BLAS级别的计算强度：

  • Level 1: O(1) - 受限于内存带宽
  • Level 2: O(n) - 部分受限于内存带宽
  • Level 3: O(n) - 可充分利用处理器性能
证明：对于n×n矩阵乘法C=AB：
  • 浮点运算：2n³次（每个元素需n次乘法和n次加法）
  • 内存访问：3n²次（读取A、B各一次，写入C一次）
  • 计算强度：2n³/(3n²) = O(n)
当n增大时，计算时间主导内存访问时间，可以更好地利用CPU缓存和流水线。

(b) BLAS Level 3 高性能原理

BLAS Level 3可以接近处理器峰值性能的关键原因：

  1. 高计算强度
    • O(n³)的计算量 vs O(n²)的内存访问量
    • 计算/内存比率为O(n)，n越大性能越好
  2. 数据重用性
    • 在C=AB中，A的每个元素被重用n次
    • B的每个元素被重用n次
    • 可以将数据保持在快速缓存中
  3. 缓存优化（Blocking/Tiling）
    • 将大矩阵分成适合L1/L2/L3缓存的小块
    • 最小化缓存缺失（cache misses）
    • 典型分块大小：32×32或64×64
  4. 向量化（SIMD）
    • 利用AVX、SSE等指令同时处理多个数据
    • 一条指令可执行4-8个浮点运算
  5. 多线程并行
    • 矩阵乘法天然可并行
    • 可在多核CPU上分配工作负载

(c) LAPACK 与 BLAS 的关系

LAPACK（Linear Algebra PACKage）是建立在BLAS之上的高级线性代数库。

层次关系：

┌─────────────────────────────────────┐
│   应用程序 / 科学计算软件              │
│   (NumPy, MATLAB, Julia, etc.)     │
└────────────────┬────────────────────┘
                 │ 调用
┌────────────────▼────────────────────┐
│          LAPACK                     │
│  高级线性代数算法：                  │
│  • 线性方程组求解 (LU, Cholesky)    │
│  • 最小二乘问题 (QR, SVD)           │
│  • 特征值问题                       │
│  • 矩阵分解                         │
└────────────────┬────────────────────┘
                 │ 调用
┌────────────────▼────────────────────┐
│          BLAS                       │
│  底层数值计算原语：                  │
│  • 向量运算 (Level 1)               │
│  • 矩阵-向量运算 (Level 2)          │
│  • 矩阵-矩阵运算 (Level 3)          │
└────────────────┬────────────────────┘
                 │
┌────────────────▼────────────────────┐
│        硬件优化实现                  │
│  (OpenBLAS, MKL, ATLAS, etc.)      │
└─────────────────────────────────────┘

主要特点：

  • 功能定位：LAPACK提供复杂算法，BLAS提供基础操作
  • 性能保证：LAPACK通过调用BLAS获得优化性能
  • 可移植性：只需替换BLAS实现即可在不同平台优化
  • 接口标准：两者都是工业标准，被广泛支持

示例：求解线性方程组Ax=b的调用链

  1. 应用程序调用LAPACK的 dgesv（求解一般线性系统）
  2. LAPACK内部执行LU分解，调用BLAS的 dgemm（矩阵乘法）
  3. BLAS库（如MKL）执行高度优化的矩阵乘法

(d) 向量化操作优于For循环的原因

在脚本语言（Python、MATLAB、R等）中，向量化操作显著优于显式for循环：

方面 向量化操作 For循环
执行层 编译的C/Fortran代码 解释执行的脚本
性能 快（10-100倍） 慢
内存访问 连续，缓存友好 可能不连续
SIMD 自动使用 不使用
多线程 可能自动并行 单线程
代码长度 简洁 冗长

性能差异的根本原因：

  1. 解释器开销：每次循环迭代都要解释Python字节码
  2. 类型检查：动态类型语言每次操作都要检查类型
  3. 函数调用：循环内的操作涉及多次函数调用
  4. 内存管理：频繁的对象创建和垃圾回收

(e) 共享内存并行（Shared Memory Parallelism）

定义：共享内存并行是指多个处理器或核心访问同一个物理内存地址空间的并行计算模型。

架构特点：

┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│  CPU 0  │  │  CPU 1  │  │  CPU 2  │  │  CPU 3  │
│ Cache   │  │ Cache   │  │ Cache   │  │ Cache   │
└────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘
     │            │            │            │
     └────────────┴────────────┴────────────┘
                  │
     ┌────────────▼─────────────┐
     │   共享内存 (Shared RAM)   │
     │                          │
     │   全局变量、堆内存        │
     └──────────────────────────┘

主要特征：

  1. 统一地址空间
    • 所有线程/进程看到相同的内存地址
    • 可以直接读写共享变量
  2. 低延迟通信
    • 通过内存直接交换数据
    • 比消息传递快得多
  3. 同步机制
    • 需要锁（locks）、信号量（semaphores）
    • 防止数据竞争（data races）
    • 原子操作（atomic operations）
  4. 缓存一致性
    • 硬件保证各CPU缓存的一致性
    • 使用MESI等协议

常用技术和库：

  • OpenMP：编译器指令，易用
  • Pthreads：POSIX线程，细粒度控制
  • C++11 threads：现代C++标准库
  • TBB (Threading Building Blocks)：Intel高级并行库

优势：

  • 编程相对简单（相比分布式内存）
  • 通信开销低
  • 适合细粒度并行

劣势：

  • 扩展性受限（通常≤数百核心）
  • 内存带宽成为瓶颈
  • 需要仔细管理同步，容易出现bug（死锁、竞态条件）

Amdahl定律（加速比上限）

定理：设程序中可并行部分占总执行时间的比例为 p，使用 n 个处理器时，理论加速比为： $$S(n) = \frac{1}{(1-p) + \frac{p}{n}}$$ 当 \(n \to \infty\) 时，最大加速比为： $$S_{\max} = \frac{1}{1-p}$$
证明：
  • 设总时间为T，串行部分时间为(1-p)T，并行部分时间为pT
  • 使用n个处理器后，并行部分时间变为pT/n
  • 总时间变为：\(T_{parallel} = (1-p)T + \frac{pT}{n}\)
  • 加速比：\(S = \frac{T}{T_{parallel}} = \frac{1}{(1-p) + p/n}\)
重要推论：
  • 如果p=0.9（90%可并行），最大加速比仅为10倍
  • 如果p=0.95（95%可并行），最大加速比仅为20倍
  • 串行瓶颈严重限制了并行加速
# Python示例：使用multiprocessing演示共享内存并行
import numpy as np
import multiprocessing as mp
from multiprocessing import Pool, Array
import time

# 示例1：并行计算数组平方和
def compute_square_sum(args):
    """计算数组一部分的平方和"""
    start, end, arr = args
    return sum(arr[start:end]**2)

def parallel_square_sum(arr, num_processes=4):
    """使用多进程并行计算"""
    n = len(arr)
    chunk_size = n // num_processes

    # 划分任务
    tasks = []
    for i in range(num_processes):
        start = i * chunk_size
        end = (i + 1) * chunk_size if i < num_processes - 1 else n
        tasks.append((start, end, arr))

    # 并行执行
    with Pool(processes=num_processes) as pool:
        results = pool.map(compute_square_sum, tasks)

    return sum(results)

# 测试

def amdahl_speedup(p, n):
    """计算Amdahl定律预测的加速比"""
    return 1 / ((1 - p) + p / n)

# 不同并行比例的加速比

num_processors = [1, 2, 4, 8, 16, 32, 64]

print("\n并行比例 | ", end="")
for n in num_processors:

    print(f"  {p*100:4.0f}%    | ", end="")
    for n in num_processors:
        speedup = amdahl_speedup(p, n)
        print(f"{speedup:6.2f} | ", end="")
    print(f"  (最大:{1/(1-p):.1f}x)")

print("\n观察：即使有无限多核心，串行部分仍然限制加速比！")

(a) LU 分解求解线性系统 - 完整步骤

给定系统：

$$A = \begin{bmatrix} 2 & 0 & 0 \\ 4 & 3 & 0 \\ 8 & 9 & 5 \end{bmatrix}, \quad b = \begin{bmatrix} 2 \\ 1 \\ 4 \end{bmatrix}$$

步骤 1：LU 分解

观察到A是下三角矩阵。我们需要找到L（单位下三角）和U（上三角）使得A = LU。

高斯消元过程：

第1步：消去第一列下方元素

  • 乘子：\(l_{21} = a_{21}/a_{11} = 4/2 = 2\)
  • 乘子：\(l_{31} = a_{31}/a_{11} = 8/2 = 4\)
  • 行2: [4 3 0] - 2×[2 0 0] = [0 3 0]
  • 行3: [8 9 5] - 4×[2 0 0] = [0 9 5]

第2步：消去第二列下方元素

  • 乘子：\(l_{32} = 9/3 = 3\)
  • 行3: [0 9 5] - 3×[0 3 0] = [0 0 5]

得到 L 和 U：

$$L = \begin{bmatrix} 1 & 0 & 0 \\ 2 & 1 & 0 \\ 4 & 3 & 1 \end{bmatrix}, \quad U = \begin{bmatrix} 2 & 0 & 0 \\ 0 & 3 & 0 \\ 0 & 0 & 5 \end{bmatrix}$$

验证：

$$LU = \begin{bmatrix} 1 & 0 & 0 \\ 2 & 1 & 0 \\ 4 & 3 & 1 \end{bmatrix} \begin{bmatrix} 2 & 0 & 0 \\ 0 & 3 & 0 \\ 0 & 0 & 5 \end{bmatrix} = \begin{bmatrix} 2 & 0 & 0 \\ 4 & 3 & 0 \\ 8 & 9 & 5 \end{bmatrix} = A \quad \checkmark$$

步骤 2：前向替换求解 Ly = b

$$\begin{bmatrix} 1 & 0 & 0 \\ 2 & 1 & 0 \\ 4 & 3 & 1 \end{bmatrix} \begin{bmatrix} y_1 \\ y_2 \\ y_3 \end{bmatrix} = \begin{bmatrix} 2 \\ 1 \\ 4 \end{bmatrix}$$

  • \(y_1 = 2\)
  • \(2y_1 + y_2 = 1 \Rightarrow y_2 = 1 - 2(2) = 1 - 4 = -3\)
  • \(4y_1 + 3y_2 + y_3 = 4 \Rightarrow y_3 = 4 - 4(2) - 3(-3) = 4 - 8 + 9 = 5\)

$$y = \begin{bmatrix} 2 \\ -3 \\ 5 \end{bmatrix}$$

步骤 3：后向替换求解 Ux = y

$$\begin{bmatrix} 2 & 0 & 0 \\ 0 & 3 & 0 \\ 0 & 0 & 5 \end{bmatrix} \begin{bmatrix} x_1 \\ x_2 \\ x_3 \end{bmatrix} = \begin{bmatrix} 2 \\ -3 \\ 5 \end{bmatrix}$$

  • \(5x_3 = 5 \Rightarrow x_3 = 1\)
  • \(3x_2 = -3 \Rightarrow x_2 = -1\)
  • \(2x_1 = 2 \Rightarrow x_1 = 1\)

最终解：

$$\boxed{x = \begin{bmatrix} 1 \\ -1 \\ 1 \end{bmatrix}}$$

验证：

$$Ax = \begin{bmatrix} 2 & 0 & 0 \\ 4 & 3 & 0 \\ 8 & 9 & 5 \end{bmatrix} \begin{bmatrix} 1 \\ -1 \\ 1 \end{bmatrix} = \begin{bmatrix} 2 \\ 4-3 \\ 8-9+5 \end{bmatrix} = \begin{bmatrix} 2 \\ 1 \\ 4 \end{bmatrix} = b \quad \checkmark$$

# 方法1：手动实现LU分解
def lu_decomposition_manual(A):
    """手动实现LU分解（不带主元）"""
    n = len(A)
    L = np.eye(n)
    U = A.copy()

    for k in range(n-1):
        for i in range(k+1, n):
            # 计算乘子
            L[i, k] = U[i, k] / U[k, k]
            # 消元
            U[i, k:] = U[i, k:] - L[i, k] * U[k, k:]

    return L, U

L_manual, U_manual = lu_decomposition_manual(A)

print(L_manual @ U_manual)
print("原矩阵 A:")
print(A)
print(f"误差: {np.linalg.norm(L_manual @ U_manual - A):.2e}")

# 前向替换求解 Ly = b
def forward_substitution(L, b):
    """前向替换"""
    n = len(b)
    y = np.zeros(n)
    for i in range(n):
        y[i] = b[i] - np.dot(L[i, :i], y[:i])
    return y

# 后向替换求解 Ux = y
def backward_substitution(U, y):
    """后向替换"""
    n = len(y)
    x = np.zeros(n)
    for i in range(n-1, -1, -1):
        x[i] = (y[i] - np.dot(U[i, i+1:], x[i+1:])) / U[i, i]
    return x

x_manual = backward_substitution(U_manual, y)

LU分解存在性和唯一性定理

定理：矩阵 \(A \in \mathbb{R}^{n \times n}\) 有LU分解 \(A = LU\)（其中L是单位下三角，U是上三角）当且仅当A的所有顺序主子式都非零。

顺序主子式：A的前k×k子矩阵的行列式，k=1,2,...,n

证明（必要性⇒）：

设 \(A = LU\)，则A的前k×k子矩阵 \(A_k = L_k U_k\)，其中 \(L_k, U_k\) 是L, U的前k×k子矩阵。

因为 \(L_k\) 是单位下三角（对角元为1），\(\det(L_k) = 1\)

因为 \(U_k\) 是上三角，\(\det(U_k) = \prod_{i=1}^k u_{ii}\)

所以 \(\det(A_k) = \det(L_k)\det(U_k) = \prod_{i=1}^k u_{ii} \neq 0\)（假设LU分解存在）

证明（充分性⇐）：

通过数学归纳法构造LU分解。若所有顺序主子式非零，高斯消元的每一步都不会遇到零主元，因此可以顺利进行直到完成。

(b) 误差分析基本概念

(i) 三个关键概念的定义

1. 相对前向误差（Relative Forward Error）

设真实值为 \(y = f(x)\)，计算值为 \(\tilde{y} = \tilde{f}(x)\)，则：

$$\text{相对前向误差} = \frac{\|\tilde{y} - y\|}{\|y\|} = \frac{\|\tilde{f}(x) - f(x)\|}{\|f(x)\|}$$

解释：衡量输出的相对误差大小。

2. 相对后向误差（Relative Backward Error）

找到最小的扰动 \(\Delta x\) 使得 \(\tilde{y} = f(x + \Delta x)\)，则：

$$\text{相对后向误差} = \min \left\{ \frac{\|\Delta x\|}{\|x\|} : \tilde{y} = f(x + \Delta x) \right\}$$

解释：要使计算结果精确，输入需要的最小相对扰动。即：计算结果是"附近某个问题"的精确解。

3. 条件数（Condition Number）

$$\kappa(f, x) = \lim_{\varepsilon \to 0} \sup_{\|\Delta x\| \leq \varepsilon \|x\|} \frac{\|f(x+\Delta x) - f(x)\|/\|f(x)\|}{\|\Delta x\|/\|x\|}$$

对于可微函数：

$$\kappa(f, x) = \frac{\|J_f(x)\| \cdot \|x\|}{\|f(x)\|}$$

其中 \(J_f(x)\) 是Jacobian矩阵。

解释：条件数衡量问题的敏感度。条件数大的问题称为"病态"（ill-conditioned）。

三者关系：

$$\text{相对前向误差} \approx \kappa(f, x) \times \text{相对后向误差}$$

这说明：条件数是误差放大因子！

(b)(ii) 后向稳定性的定义

定义：算法 \(\tilde{f}\) 称为后向稳定的（Backward Stable），如果对于任何输入 \(x\)，计算结果 \(\tilde{y} = \tilde{f}(x)\) 满足：

$$\tilde{y} = f(x + \Delta x)$$

其中相对后向误差为：

$$\frac{\|\Delta x\|}{\|x\|} = O(\varepsilon_{mach})$$

\(\varepsilon_{mach}\) 是机器精度（machine epsilon）。

直观理解：

  • 后向稳定 = 计算结果是"轻微扰动输入"的精确解
  • 扰动大小 ≈ 机器精度 = 数据本身的不确定性
  • 这是数值算法能达到的最好情况

重要性：

后向稳定算法保证：

$$\text{相对前向误差} \leq \kappa(f, x) \cdot O(\varepsilon_{mach})$$

即：误差完全由问题的条件数决定，算法本身不引入额外误差。

例子：

  • 后向稳定：Householder QR分解、带主元的LU分解
  • 不保证后向稳定：不带主元的LU分解、Gram-Schmidt正交化（经典版本）

(c) 矩阵扰动的误差界 - 详细证明

给定：

  • \(Ax = b\)（原系统）
  • \(\tilde{A}\tilde{x} = b\)（扰动系统），其中 \(\tilde{A} = A + \delta A\)

目标：证明 \(\displaystyle\frac{\|x - \tilde{x}\|}{\|x\|} \leq \kappa(A) \frac{\|\delta A\|}{\|A\|}\)

证明：

步骤 1：建立方程

从两个系统：

  • \(Ax = b\)
  • \((A + \delta A)\tilde{x} = b\)

展开第二个方程：

$$A\tilde{x} + \delta A \tilde{x} = b$$

两式相减：

$$A\tilde{x} + \delta A \tilde{x} - Ax = 0$$

$$A(\tilde{x} - x) = -\delta A \tilde{x}$$

步骤 2：求解误差

$$\tilde{x} - x = -A^{-1} \delta A \tilde{x}$$

步骤 3：取范数

$$\|\tilde{x} - x\| = \|A^{-1} \delta A \tilde{x}\|$$

使用范数的次可乘性 \(\|AB\| \leq \|A\| \|B\|\)：

$$\|\tilde{x} - x\| \leq \|A^{-1}\| \cdot \|\delta A\| \cdot \|\tilde{x}\|$$

步骤 4：除以 \(\|x\|\)

$$\frac{\|\tilde{x} - x\|}{\|x\|} \leq \|A^{-1}\| \cdot \|\delta A\| \cdot \frac{\|\tilde{x}\|}{\|x\|}$$

步骤 5：一阶近似

当扰动足够小时，\(\|\tilde{x}\| \approx \|x\|\)（一阶近似），因此：

$$\frac{\|\tilde{x}\|}{\|x\|} \approx 1$$

代入得：

$$\frac{\|\tilde{x} - x\|}{\|x\|} \leq \|A^{-1}\| \cdot \|\delta A\|$$

步骤 6：引入条件数

从 \(Ax = b\) 得到 \(\|b\| \leq \|A\| \|x\|\)，即 \(\|x\| \geq \|b\|/\|A\|\)

因此：

$$\frac{\|\tilde{x} - x\|}{\|x\|} \leq \|A^{-1}\| \cdot \|\delta A\| = \|A^{-1}\| \cdot \|A\| \cdot \frac{\|\delta A\|}{\|A\|}$$

$$= \kappa(A) \cdot \frac{\|\delta A\|}{\|A\|}$$

结论：

$$\boxed{\frac{\|x - \tilde{x}\|}{\|x\|} \leq \kappa(A) \frac{\|\delta A\|}{\|A\|}}$$

物理意义：

  • 左边：解的相对误差
  • 右边：\(\kappa(A)\)（放大因子）× 矩阵的相对扰动
  • 条件数\(\kappa(A)\)越大，解对矩阵扰动越敏感

条件数的性质与计算

基本性质：
  • \(\kappa(A) \geq 1\) （任何范数）
  • \(\kappa(\alpha A) = \kappa(A)\) 对任意 \(\alpha \neq 0\)
  • \(\kappa(A) = \kappa(A^{-1})\)
  • 对于正交矩阵Q：\(\kappa_2(Q) = 1\) （最优）
  • \(\kappa(AB) \leq \kappa(A) \kappa(B)\)
2-范数条件数（最常用）：

$$\kappa_2(A) = \|A\|_2 \|A^{-1}\|_2 = \frac{\sigma_{\max}(A)}{\sigma_{\min}(A)}$$

其中 \(\sigma_{\max}, \sigma_{\min}\) 是最大和最小奇异值。

证明 \(\kappa_2(A) = \sigma_{\max}/\sigma_{\min}\)：

对于2-范数，\(\|A\|_2 = \sigma_{\max}(A)\)（最大奇异值）

而 \(\|A^{-1}\|_2 = 1/\sigma_{\min}(A)\)（A的最小奇异值的倒数）

因此 \(\kappa_2(A) = \sigma_{\max}(A) \cdot \frac{1}{\sigma_{\min}(A)} = \frac{\sigma_{\max}(A)}{\sigma_{\min}(A)}\)

def test_condition_number(A, b, delta_A_scale):
    """测试条件数对误差的影响"""
    # 原系统的解
    x_true = np.linalg.solve(A, b)

    # 添加扰动
    delta_A = delta_A_scale * np.random.randn(*A.shape)
    A_perturbed = A + delta_A

    # 扰动系统的解
    x_perturbed = np.linalg.solve(A_perturbed, b)

    # 计算误差和条件数
    relative_error = np.linalg.norm(x_true - x_perturbed) / np.linalg.norm(x_true)
    condition_number = np.linalg.cond(A)
    relative_perturbation = np.linalg.norm(delta_A) / np.linalg.norm(A)

    # 理论上界
    upper_bound = condition_number * relative_perturbation

    return relative_error, upper_bound, condition_number, relative_perturbation
